Back-end (Rails 8 API-only)

Versioned namespace, every endpoint documented by an rswag request spec that emits swagger/v1/swagger.yaml as a side effect. Auth is a single admin user with a stateless JWT — login returns a Bearer token, the SPA sends it in the Authorization header on every write. No cookies, no CSRF middleware: the header is not auto-attached cross-origin, so CSRF is structurally out of reach.

Snippets I want to point at

1. Auth-aware scope on the API base controller

Public callers see only published records. The same URL returns drafts to an authenticated admin. One method on BaseController, three controllers reuse it. 

# app/controllers/api/v1/base_controller.rb
def visible_scope_for(model_class)
  current_user ? model_class.order(:position) : model_class.published
end

# usage
def index
  render json: {{ '{' }} projects: visible_scope_for(Project).map {{ '{' }} |p| serialize(p) {{ '}' }} {{ '}' }}
end

2. The JWT auth interceptor

One interceptor wires Authorization: Bearer ... onto every outbound request when a token is present in TokenService. No cookies, no withCredentials, no CSRF surface — the header is not auto-attached cross-origin, so attackers can't ride it. 

// core/interceptors/auth.interceptor.ts
export const authInterceptor: HttpInterceptorFn = (req, next) => {{ '{' }}
  const token = inject(TokenService).get();
  return token
    ? next(req.clone({{ '{' }} setHeaders: {{ '{' }} Authorization: `Bearer ${{ '$' }}{{ '{' }}token{{ '}' }}` {{ '}' }} {{ '}' }}))
    : next(req);
{{ '}' }};

3. ThemeService — one effect syncs three things

signal() for state, effect() for side-effects. Every theme change updates my own custom-property class, Bootstrap's data-bs-theme attribute, and localStorage in one pass. SSR-safe via isPlatformBrowser. 

constructor() {{ '{' }}
  effect(() => {{ '{' }}
    const t = this._theme();
    const root = this.doc.documentElement;
    root.classList.remove('theme-light', 'theme-dark');
    root.classList.add(`theme-${{ '$' }}{{ '{' }}t{{ '}' }}`);
    root.setAttribute('data-bs-theme', t);
    if (isPlatformBrowser(this.platformId)) {{ '{' }}
      localStorage.setItem('theme', t);
    {{ '}' }}
  {{ '}' }});
{{ '}' }}

4. authGuard — return a UrlTree, not a navigate()

Functional guards. Returning a UrlTree lets Angular handle the redirect itself, which means no race conditions with the route's own navigation and no imperative router.navigate() hiding inside a guard. 

// core/guards/auth.guard.ts
export const authGuard: CanActivateFn = () => {{ '{' }}
  const auth = inject(AuthService);
  const router = inject(Router);
  return auth.isAuthenticated()
    ? true
    : router.createUrlTree(['/admin/login']);
{{ '}' }};

5. rswag — the request spec IS the API doc

Every endpoint has a request spec in the rswag DSL. The same file runs assertions and emits swagger/v1/swagger.yaml. "Document the API later" is no longer a thing on this codebase. 

# spec/requests/api/v1/projects_spec.rb
path "/api/v1/projects" do
  get("List projects (published only when public; all when authenticated)") do
    tags "Projects"
    produces "application/json"

    response(200, "ok") do
      schema type: :object,
             properties: {{ '{' }} projects: {{ '{' }} type: :array, items: PROJECT_SCHEMA {{ '}' }} {{ '}' }},
             required: %w[projects]

      before do
        create(:project, title: "Public", published: true)
        create(:project, title: "Hidden Draft", published: false)
      end

      run_test! do |response|
        titles = JSON.parse(response.body)["projects"].map {{ '{' }} |p| p["title"] {{ '}' }}
        expect(titles).to include("Public")
        expect(titles).not_to include("Hidden Draft")
      end
    end
  end
end

6. Stateless JWT auth — one service, two methods

Login signs a JWT with secret_key_base; every subsequent request decodes the Authorization: Bearer header in the Authentication concern. No sessions, no cookies, no SameSite/Secure dance to keep dev and prod in sync. 

# app/services/json_web_token.rb
class JsonWebToken
  ALGORITHM = "HS256"
  DEFAULT_EXP = 7.days

  def self.encode(payload, exp: DEFAULT_EXP.from_now)
    JWT.encode(payload.merge(exp: exp.to_i), secret, ALGORITHM)
  end

  def self.decode(token)
    JWT.decode(token, secret, true, algorithm: ALGORITHM).first
  rescue JWT::DecodeError, JWT::ExpiredSignature
    nil
  end

  def self.secret = Rails.application.secret_key_base
end

7. CORS — narrow allowlist, no credentials needed

No CSRF middleware. With JWT in the Authorization header (not a cookie), browsers don't auto-attach it cross-origin — CSRF is structurally impossible. The CORS allowlist exists to limit who can even reach the API. 

# config/initializers/cors.rb
Rails.application.config.middleware.insert_before 0, Rack::Cors do
  allow do
    origins "http://localhost:4200", "http://127.0.0.1:4200"

    resource "*",
      headers: :any,
      expose:  %w[Authorization],
      methods: %i[get post put patch delete options head]
  end
end

9. ActiveRecord normalizes: (Rails 7.1+)

Built-in attribute normalization. No before_validation callback, no custom setter, no "is the email lower-cased on this read or that one" inconsistency. 

# app/models/contact_message.rb
class ContactMessage < ApplicationRecord
  normalizes :name, :message, with: ->(v) {{ '{' }} v.strip {{ '}' }}
  normalizes :email,           with: ->(v) {{ '{' }} v.strip.downcase {{ '}' }}

  validates :name, :message, presence: true

10. Theme tokens — CSS custom properties, not Sass variables

Sass variables resolve at build time and don't react to a class change. CSS custom properties resolve at runtime. Switching one class on <html> recolors every component without a single runtime style recalc in JS. 

// styles.scss
:root, .theme-light {{ '{' }}
  --bg:          #ffffff;
  --bg-surface:  #f8f9fa;
  --fg:          #111111;
  --accent:      #4b9cd3;
{{ '}' }}

.theme-dark {{ '{' }}
  --bg:          #18181b;
  --bg-surface:  #27272a;
  --fg:          #fafafa;
  --accent:      #4b9cd3;
{{ '}' }}

// any component
.card {{ '{' }} background: var(--bg-surface); color: var(--fg); {{ '}' }}

11. ParallaxDirective — afterNextRender, signal inputs, reduced-motion

Browser-only setup without an isPlatformBrowser guard: afterNextRender only runs in the browser, so the scroll listener never registers during SSR. Signal input() with an alias keeps the public attribute terse, rAF throttling caps work at one update per frame, and the whole effect opts out under prefers-reduced-motion: reduce. 

// shared/directives/parallax.directive.ts
@Directive({{ '{' }} selector: '[appParallax]', standalone: true {{ '}' }})
export class ParallaxDirective {{ '{' }}
  readonly factor = input(0.3, {{ '{' }} alias: 'appParallax' {{ '}' }});

  constructor() {{ '{' }}
    afterNextRender(() => {{ '{' }}
      if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) return;
      let ticking = false;
      const update = () => {{ '{' }}
        // translate3d clamped to ±factor * viewportH (no overflow past section)
        ticking = false;
      {{ '}' }};
      window.addEventListener('scroll', () => {{ '{' }}
        if (!ticking) {{ '{' }} ticking = true; requestAnimationFrame(update); {{ '}' }}
      {{ '}' }}, {{ '{' }} passive: true {{ '}' }});
    {{ '}' }});
  {{ '}' }}
{{ '}' }}

12. Project — from interface to class with a time_ago field

The frontend's Project graduated from a structural interface to a real class so it can carry behavior. ProjectData still describes the raw API shape; the class wraps it, exposes user-set project_start / project_end dates from the backend, and computes time_ago once in the constructor against project_end. Everything is readonly — the value is deterministic at construction, so a stored field is cheaper than a getter and reads the same in templates: {{ '{' }}{{ '{' }} project.time_ago {{ '}' }}{{ '}' }}. The ProjectsService maps every API response through new Project(p) so consumers always receive real instances. 

// core/models/project.model.ts
export interface ProjectData {{ '{' }}
  id: number;
  title: string;
  // …
  project_start: string | null;   // 'YYYY-MM-DD' from the API
  project_end:   string | null;
{{ '}' }}

export class Project {{ '{' }}
  readonly id: number;
  readonly title: string;
  // …
  readonly project_start: string | null;
  readonly project_end: string | null;
  readonly time_ago: string;

  constructor(data: ProjectData, now: Date = new Date()) {{ '{' }}
    this.id = data.id;
    this.title = data.title;
    // …copy the rest…
    this.project_start = data.project_start;
    this.project_end   = data.project_end;
    this.time_ago = Project.computeTimeAgo(now, data.project_end);
  {{ '}' }}

  private static computeTimeAgo(now: Date, project_end: string | null): string {{ '{' }}
    if (!project_end) return '';
    const days = Math.floor((now.getTime() - new Date(project_end).getTime()) / 86_400_000);
    if (days < 1) return 'today';
    if (days < 7) return days + ' day' + (days === 1 ? '' : 's') + ' ago';
    // …walks days → weeks → months → years and picks the right unit
  {{ '}' }}
{{ '}' }}

// core/services/projects.service.ts
list() {{ '{' }}
  return this.api.get<{{ '{' }} projects: ProjectData[] {{ '}' }}>('projects').pipe(
    map((res) => ({{ '{' }} projects: res.projects.map((p) => new Project(p)) {{ '}' }})),
  );
{{ '}' }}
